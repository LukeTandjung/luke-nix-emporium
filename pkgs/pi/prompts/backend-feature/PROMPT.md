# Backend Feature Implementation

Before starting, verify:
- [ ] Acceptance criteria have been provided
- [ ] A list of base and edge cases to test against has been provided

If either is missing, ask the user to provide them before proceeding.

## Requirements

1. **Unit testing**: All code written must be unit tested and must pass all defined cases.
2. **Test coverage**: All code written for the feature must have at least 80% test coverage.
3. **Test cases**: Cover both the base cases and edge cases provided by the user.
